Implementation Guide

Step-by-step instructions to fix the two icon issues and add a dedicated notification icon to the companion app.

🎯

What Needs to Change (2 files + 1 new file)

1. NEW drawable/ic_notification.xml — monochrome vector notification icon
2. UPDATE drawable/ic_launcher_background.xml — fix gradient colors (if going with Alt 1)
3. UPDATE service/NeuralBridgeAccessibilityService.kt — set .setSmallIcon(R.drawable.ic_notification)

~30 min total effort No proto changes needed No Gradle changes needed
1️⃣
Create ic_notification.xml (choose one variant)
android/app/src/main/res/drawable/ic_notification.xml

Create a new vector drawable. Android uses the alpha channel — the color you set here is ignored on API 21+; use android:tint or let the system colorize it. Background must be transparent.

2️⃣
Fix the launcher gradient (Alt 1 only)
android/app/src/main/res/drawable/ic_launcher_background.xml

Update the gradient stop colors from the ocean-depths palette (#004D61 → #1A237E) to the brand palette (#5E72E4 → #825EE4) to match the source SVG.

<!-- BEFORE -->
<item android:offset="0" android:color="#004D61"/>
<item android:offset="1" android:color="#1A237E"/>

<!-- AFTER (matches wave-bridge.svg source) -->
<item android:offset="0" android:color="#5E72E4"/>
<item android:offset="1" android:color="#825EE4"/>

Also update values/ic_launcher_background.xml fallback color from #004D61 to #5E72E4 for devices without gradient support.

3️⃣
Update the foreground service notification
android/app/src/main/kotlin/com/neuralbridge/companion/service/NeuralBridgeAccessibilityService.kt

Find the NotificationCompat.Builder call for the foreground service notification and replace the icon reference.

// BEFORE (using launcher icon — wrong!)
NotificationCompat.Builder(this, CHANNEL_ID)
    .setSmallIcon(R.mipmap.ic_launcher)   // ← colored icon, violates MD3
    .setContentTitle("NeuralBridge")
    ...

// AFTER
NotificationCompat.Builder(this, CHANNEL_ID)
    .setSmallIcon(R.drawable.ic_notification)  // ← monochrome vector ✓
    .setContentTitle("NeuralBridge")
    ...
4️⃣
Check NotificationListener (if it posts notifications)
android/app/src/main/kotlin/com/neuralbridge/companion/notification/NotificationListener.kt

If the NotificationListener posts any of its own notifications, apply the same fix. Search for any setSmallIcon(R.mipmap.*) references and replace with R.drawable.ic_notification.

grep -rn "setSmallIcon\|ic_launcher" android/app/src/main/kotlin/
5️⃣
Rebuild and verify

Build the APK and install on device. Check: (1) launcher icon shows correct gradient, (2) notification in status bar shows monochrome icon, (3) adaptive icon looks correct in both square and round masks.

cd android ./gradlew assembleDebug adb install -r app/build/outputs/apk/debug/app-debug.apk # Trigger the foreground service notification: adb shell settings put secure enabled_accessibility_services \ com.neuralbridge.companion/.NeuralBridgeAccessibilityService
Verify the status bar icon with: long-press notification → "App info" to confirm it uses the new monochrome icon. If still showing color, clean build: ./gradlew clean assembleDebug

Optional — Regenerate PNG fallbacks

The launcher PNG files in mipmap-*/ are fallbacks for devices that don't support adaptive icons (API < 26). If you update the launcher SVG, regenerate them using the existing script:

cd android/design # Requires ImageMagick ./generate-icons.sh # Output: mipmap-mdpi/hdpi/xhdpi/xxhdpi/xxxhdpi PNG files

The adaptive icon XMLs in mipmap-anydpi-v26/ don't need changes — they already reference ic_launcher_background and ic_launcher_foreground which you updated above.
